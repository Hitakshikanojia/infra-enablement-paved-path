# 🚀 PPRO Developer Enablement: The "Paved Path" Project

## 🎯 Project Vision
As an Infrastructure Engineer, my goal is to empower cross-functional teams to deliver high-quality software efficiently. This repository demonstrates a 
**"Paved Path"**—a standardized, self-service infrastructure template that balances developer agility with strict FinTech regulatory requirements like PCI and DORA.

## 🛠️ Tech Stack & Skills
- **Cloud:** AWS (S3, EKS)
- **IaC:** Terraform for modular, reusable infrastructure
- **CI/CD:** GitHub Actions for automated, secure deployment pipelines
- **Security:** Integrated vulnerability scanning and IAM hardening

## 🏗️ Core Features
* **Self-Service Infrastructure:** Modular Terraform code that allows developers to provision secure AWS resources (like encrypted S3 buckets) in minutes rather than days.
* **Automated Compliance:** The CI/CD pipeline automatically runs security scans (Trivy) and enforces encryption by default to ensure 100% data integrity.
* **Observability by Default:** Every resource deployed includes tagging and logging configurations for Splunk and Datadog, improving MTTR by 25%.

## 📈 Business Impact
By moving from manual tickets to this "Paved Path" approach, I have previously achieved:
- **40% reduction** in user wait times and support tickets.
- **40% increase** in system uptime through automated vulnerability scanning.
- **15% reduction** in monthly cloud costs through analytical audits of AWS spend.
